WordCamp Bogotá 2018

Taller de introducción al desarrollo de plugins

A pocas horas del WordCamp Bogotá 2018, queremos invitarlos a verificar que cuentan en sus equipos con las herramientas que necesitaremos durante el taller para crear nuestro primer plugin de WordPress.

Necesitarán:

1. Un buen editor de texto

Idealmente uno que les permita escribir código PHP y HTML cómodamente. Recomendamos Atom o Sublime Text, ambos disponibles de manera gratuita y para todos los sistemas operativos.

2. Un entorno local de WordPress (máquina virtual)

Para quienes no deseen encargarse de instalar y configurar un servidor web local con WordPress, queremos ofrecerles una alternativa que les permitiría tener en su equipo una máquina virtual pre-configurada con todo lo que necesitan para asistir al taller.

Dentro de las ventajas de la máquina virtual está el que no afectarán el software de su equipo y, una vez terminado el taller, pueden removerla sin problemas si lo desean, o seguirla utilizando para otros proyectos. También funcionaría bajo todos los sistemas operativos sin diferencias.

Para hacer uso de esta alternativa deberán:

  1. Descargar e instalar VirtualBox
  2. Descargar e instalar Vagrant
  3. Descargar y descomprimir el archivo de entorno local que hemos preparado, en un directorio de su preferencia.
  4. Ubicarse en el directorio recién descomprimido utilizando la terminal (macOS, Linux) o la línea de comandos (Windows) y ejecutar el comando: vagrant up. Esto iniciará el proceso de configuración de la máquina virtual automáticamente.
  5. Acceder a tu nueva instalación de WordPress en la URL http://192.168.25.4/.

2. Acceso a un servidor local

Para probar el código que vayamos escribiendo será necesario contar con una instalación local de WordPress, lo que usualmente involucra instalar y configurar un servidor web (Apache, NGINX, etc.), un servidor de base de datos MySQL o MariaDB y el módulo de PHP aplicable.

Si bien este proceso puede realizarse manualmente, existen paquetes instalables como lo son AMPPS, XAMPP o MAMP que se encargan de todo por nosotros. En el caso particular de sistemas operativos basados en Linux, la instalación del software requerido puede hacerse mediante el manejador de paquetes de la distribución (por ejemplo, en Fedora o en Ubuntu).

3. Una instalación de WordPress local

Una vez tengamos correctamente configurado nuestro servidor local, instalar WordPress es un proceso sencillo, que normalmente se limita a descargar y descomprimir la última versión de WordPress en un directorio del servidor y dejar que WordPress se encargue del resto de la configuración e instalación.
